# MYMS Design System

## Brand intent

MYMS should look like a premium SaaS product built for real commercial pressure.

The visual system must communicate:

- trust
- control
- precision
- product quality
- modernity without gimmicks

## Color tokens

- black: `#000000`
- deep navy: `#050A12`
- dark blue: `#09111D`
- electric blue: `#2D7FF9`
- soft blue: `#7AB8FF`
- white: `#FFFFFF`
- muted text: `#A7B0BE`
- border: `rgba(255,255,255,0.12)`
- glass: `rgba(255,255,255,0.06)`

## Typography

- family: Inter
- headings: tight tracking, high contrast, compact line-height
- body: clean, calm, legible

## Surfaces

- use dark layered backgrounds
- glass should be restrained and functional
- cards should feel like product surfaces, not floating toys
- blue glow should be subtle and localized

## Motion

- use GSAP for premium reveal timing
- use Framer Motion for lightweight UI entrance patterns when appropriate
- use Three.js for depth and product-world atmosphere
- motion should be elegant, not loud

## Layout

- generous spacing
- large readable headlines
- clear CTA hierarchy
- strong mobile stacking behavior

## Forbidden visual moves

- purple gradients
- candy colors
- excessive blur everywhere
- huge glow blooms
- gaming HUD aesthetics
- fake cyberpunk visuals

## Product illustration logic

When showing MYMS, present modules such as:

- Landing
- Turnos
- CRM
- WhatsApp
- Panel
- Clientes
- Consultas

These are not decorative labels. They represent the system architecture.
